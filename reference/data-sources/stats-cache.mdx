## Overview

The `stats-cache.json` file contains pre-computed analytics about your Claude Code usage. This cache file aggregates data from all sessions to provide quick access to statistics, daily activity patterns, model-specific token usage, and cost estimates.

## File Location

```
~/.claude/stats-cache.json
```

## File Format

JSON format containing aggregated statistics across all sessions.

## Data Structure

### StatsCache Interface

```typescript
export interface StatsCache {
  totalSessions: number;
  totalMessages: number;
  firstSessionDate: string;
  dailyActivity: DailyActivity[];
  dailyModelTokens: DailyModelTokens[];
  modelUsage: Record<string, ModelUsage>;
  hourCounts: Record<string, number>;
  longestSession: {
    sessionId: string;
    duration: number;
    messageCount: number;
    timestamp: string;
  };
  version?: string;
  lastComputedDate?: string;
}
```

<ResponseField name="totalSessions" type="number" required>
  Total number of coding sessions across all time
</ResponseField>

<ResponseField name="totalMessages" type="number" required>
  Total number of messages (user + assistant) sent
</ResponseField>

<ResponseField name="firstSessionDate" type="string" required>
  ISO timestamp of the first recorded session
</ResponseField>

<ResponseField name="dailyActivity" type="DailyActivity[]" required>
  Array of daily activity metrics. See [DailyActivity structure](#dailyactivity-structure) below.
</ResponseField>

<ResponseField name="dailyModelTokens" type="DailyModelTokens[]" required>
  Daily token usage broken down by model. See [DailyModelTokens structure](#dailymodeltokens-structure) below.
</ResponseField>

<ResponseField name="modelUsage" type="Record<string, ModelUsage>" required>
  Aggregate token usage and costs per model. Keys are model names. See [ModelUsage structure](#modelusage-structure) below.
</ResponseField>

<ResponseField name="hourCounts" type="Record<string, number>" required>
  Message count by hour of day (0-23). Keys are hour strings.
</ResponseField>

<ResponseField name="longestSession" type="object" required>
  Metadata about your longest coding session
  
  <Expandable title="properties">
    <ResponseField name="sessionId" type="string">
      Unique identifier for the session
    </ResponseField>
    <ResponseField name="duration" type="number">
      Duration in minutes
    </ResponseField>
    <ResponseField name="messageCount" type="number">
      Total messages in the session
    </ResponseField>
    <ResponseField name="timestamp" type="string">
      ISO timestamp when the session started
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="version" type="string">
  Version of the stats cache schema
</ResponseField>

<ResponseField name="lastComputedDate" type="string">
  ISO timestamp of when the cache was last computed
</ResponseField>

### ModelUsage Structure

```typescript
export interface ModelUsage {
  inputTokens: number;
  outputTokens: number;
  cacheReadInputTokens: number;

```

<ResponseField name="inputTokens" type="number" required>
  Total input tokens consumed by this model
</ResponseField>

<ResponseField name="outputTokens" type="number" required>
  Total output tokens generated by this model
</ResponseField>

<ResponseField name="cacheReadInputTokens" type="number" required>
  Tokens read from prompt cache (cheaper than regular input)
</ResponseField>

<ResponseField name="cacheCreationInputTokens" type="number" required>

</ResponseField>

<ResponseField name="webSearchRequests" type="number" required>

</ResponseField>

<ResponseField name="costUSD" type="number">
  Estimated cost in USD for this model's usage
</ResponseField>

<ResponseField name="contextWindow" type="number">
  Maximum context window size for this model
</ResponseField>

<ResponseField name="maxOutputTokens" type="number">
  Maximum output token limit for this model
</ResponseField>

## Loading Function

From `src/lib/load-data.ts`:

```typescript
const CLAUDE_DIR = path.join(process.env.HOME || "~", ".claude");

export function loadStatsCache(): StatsCache | null {
  try {
    const raw = fs.readFileSync(path.join(CLAUDE_DIR, "stats-cache.json"), "utf-8");
    return JSON.parse(raw);
  } catch {
    return null;
  }
}
```

<Note>
  Returns `null` if the file doesn't exist or can't be parsed. This is normal for new installations.
</Note>

## Usage in Export Script

From `scripts/export.mjs`:

```javascript
// 1. Stats cache
let stats = null;
try {
  stats = JSON.parse(fs.readFileSync(path.join(CLAUDE_DIR, "stats-cache.json"), "utf-8"));
  console.log("  stats-cache.json ✓");
} catch {
  console.log("  stats-cache.json ✗ (not found)");
}

// Include in export
const data = {
  stats,
  sessions,
  history,
  memories,
  account: Object.keys(account).length > 0 ? account : undefined,
  exportedAt: new Date().toISOString(),
};
fs.writeFileSync(OUTPUT, JSON.stringify(data));
```

The export script:
1. Attempts to read `stats-cache.json`
2. Logs success or failure
3. Includes the parsed stats (or `null`) in the export bundle

## Use Cases

- **Activity Heatmap**: `dailyActivity` powers the GitHub-style contribution grid
- **Model Breakdown**: `modelUsage` displays pie charts and cost tables
- **Daily Token Chart**: `dailyModelTokens` shows stacked area charts of token consumption
- **Hour Distribution**: `hourCounts` reveals when you code most
- **Summary Stats**: `totalSessions`, `totalMessages`, and `longestSession` appear in stat cards
